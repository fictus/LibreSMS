Four tabs.
Full control.

Dashboard, Controls, Logs, and Test — everything you need to run and monitor your gateway.

01

Dashboard

Your live command center. See the gateway status, LAN address, received/sent message counters, webhook delivery stats, all HTTP endpoint URLs, and session uptime at a glance.

  • Green dot = gateway running
  • Live message counters
  • All endpoint URLs listed
  • Uptime clock
02

Controls

Configure every aspect of the gateway in one place — webhook destination, shared secret, HTTP listen port, SMS/MMS forwarding toggles, and auto-start on boot.

  • Webhook URL + secret
  • Forward SMS / MMS toggles
  • Custom listen port
  • Auto-start on boot
03

Logs

A scrolling, color-coded activity log updated in real time. Every HTTP request, webhook delivery, incoming message, and error is captured with a precise timestamp.

  • INFO / SUCCESS / WARN / ERROR
  • Timestamped entries
  • Up to 500 log entries
  • Live indicator
04

Test Console

Send a real SMS or MMS directly from the device without touching any external tooling. Also fire a test webhook to verify your server is receiving payloads correctly.

  • Send SMS by phone number
  • Send MMS with image URL
  • Fire test webhooks
  • No external tools needed

Everything you need.
Nothing you don't.

Built with .NET MAUI for Android. Runs a local HTTP server on your device and stays alive in the background.

📤

Send SMS & MMS

Fire messages via HTTP GET or POST from any device on your network. MMS supports multiple images as base64 or URL.

📥

Real-time Webhooks

Incoming SMS/MMS are forwarded instantly to your configured webhook URL, including full attachment data for MMS.

📬

Read Inbox via API

Query unread messages from the device inbox at any time through the /getmessages endpoint.

🔒

Webhook Secrets

Authenticate deliveries with a shared secret token. Your server can reject unauthorized webhook calls.

⚡

Foreground Service

A persistent foreground service with wake lock keeps the gateway alive even with the screen off.

🔄

Auto-start on Boot

Optionally start the gateway automatically every time the device powers on. Always available.

🖥

Live Dashboard

Monitor message stats, uptime, sent/received counts, and all endpoint URLs from the in-app dashboard.

🧪

Built-in Test Console

Send real messages and fire test webhooks directly from within the app — no external tooling required.

Simple REST endpoints.
Works from anywhere.

All endpoints accept GET and POST. POST supports JSON body or URL-encoded form data.

GET /sendsms Send an SMS
GET /sendmms Send MMS with images
GET /getmessages Read unread inbox
GET /status Gateway status & stats
GET /health Health check
Example — send MMS with image URL HTTP
GET http://<device-ip>:8686/sendmms
  ?to=+15551234567
  &message=Look+at+this
  &image_url=https://example.com/photo.jpg

// Response
{
  "success": true,
  "message": "MMS sent to +15551234567"

Download the .APK

Up in four steps.

Requires an Android device with an active SIM, .NET 10 SDK, and Visual Studio 2026 with the MAUI workload.

01

Clone the repository

git clone https://github.com/fictus/LibreSMS.git then open LibreSMS.sln in Visual Studio. Install the MAUI workload if prompted.

02

Build & deploy to your Android device

Connect your device via USB with developer mode and USB debugging enabled. Press Run in Visual Studio or use dotnet run --project LibreSMS/LibreSMS.csproj -f net10.0-android.

03

Set LibreSMS as default SMS app

Required to receive incoming messages. Tap SET AS DEFAULT on the Dashboard tab. Without this, only outbound sending works.

04

Configure and start the gateway

Set your Webhook URL and listen port (default 8686) in the Controls tab. Tap START GATEWAY — the status dot turns green when ready.
